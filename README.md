# MealsApp

A full-stack Flutter mobile application for discovering and managing meal recipes. Users authenticate, browse categorised meals, search by name or ingredient, save favourites, build a shopping list, and leave reviews — all backed by Firebase.

---

## Tech Stack

| Layer    | Technology                                      |
|----------|-------------------------------------------------|
| Frontend | Flutter 3, Material 3, Dart                     |
| Auth     | Firebase Authentication (Email & Password)      |
| Database | Cloud Firestore (NoSQL)                         |
| Fonts    | Google Fonts (Lato, Playfair Display)           |

---

## Features

### Authentication
- Register and log in with email and password
- Session persists across app restarts via Firebase Auth state stream
- Auth gate in `main.dart` — unauthenticated users see the login screen, authenticated users go straight to the app

### Browsing
- Colour-coded category grid — 10 cuisine categories
- Tap a category to see all meals within it, filtered by active dietary preferences
- Hero animation from meal card to detail screen

### Search
- Live search by meal name or ingredient across all Firestore meals
- Works alongside active dietary filters

### Meal Details
- Full recipe page with image, meta chips (duration, complexity, affordability)
- Dietary badges — Gluten-Free, Lactose-Free, Vegan, Vegetarian
- Numbered step-by-step instructions

### Favourites
- Heart icon on detail screen toggles favourite status
- Persisted per user in Firestore — survives app restarts
- Dedicated Favourites tab with real-time sync

### Shopping List
- Add all ingredients from any meal in one tap
- Check off items as you shop
- Swipe to delete individual items
- Clear all checked items at once

### Reviews & Ratings
- Star rating input (1–5) with optional comment
- One review per user per meal
- Average rating recalculated in Firestore on every submission
- Reviews displayed on the detail screen with user avatar and date

### Dietary Filters
- Filter drawer accessible from the Categories tab
- Toggle Gluten-Free, Lactose-Free, Vegan, Vegetarian
- Active filter indicator dot on the filter icon
- Filters apply across Categories, Search, and Favourites simultaneously

## Getting Started

### Prerequisites

- [Flutter SDK](https://flutter.dev/docs/get-started/install) 3.x or later
- [Node.js](https://nodejs.org/) (for Firebase CLI)
- Android Studio with an Android emulator configured
- A Firebase account (free)

---

### 1. Create a new Flutter project

```bash
flutter create meals
cd meals
```

---

### 2. Add dependencies

```bash
flutter pub add firebase_core
flutter pub add firebase_auth
flutter pub add cloud_firestore
flutter pub add google_fonts
```

---

### 3. Replace the lib/ folder

Delete the generated `lib/` folder and paste in the one from this repository. Your structure should match the tree above exactly.

---

### 4. Firebase — Create a project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Click **Add project** → name it → click **Create project**

---

### 5. Firebase — Enable Authentication

1. In the Firebase console, click **Authentication** → **Get started**
2. Under **Sign-in method**, click **Email/Password** → toggle **Enable** → **Save**

---

### 6. Firebase — Enable Firestore

1. Click **Firestore Database** → **Create database**
2. Select **Start in test mode**
3. Choose any region → **Enable**

---

### 7. Install Firebase CLI and FlutterFire CLI

```bash
npm install -g firebase-tools
firebase login
dart pub global activate flutterfire_cli
```

If `flutterfire` is not recognised after install, add the Pub cache to your PATH:

**Windows (PowerShell — current session):**
```powershell
$env:PATH += ";C:\Users\YOUR_USERNAME\AppData\Local\Pub\Cache\bin"
```

**Windows (permanent):** Add `C:\Users\YOUR_USERNAME\AppData\Local\Pub\Cache\bin` to your system PATH via Environment Variables settings.

**macOS/Linux:**
```bash
export PATH="$PATH":"$HOME/.pub-cache/bin"
```

---

### 8. Connect Flutter to Firebase

Inside your project folder:

```bash
flutterfire configure
```

- Select your Firebase project
- Select **Android** as the platform (Space to select, Enter to confirm)

This generates `lib/firebase_options.dart` automatically — do not create this file manually.

---

### 9. Run the app

Start your Android emulator from Android Studio, then:

```bash
flutter run
```

On first launch the app automatically seeds all 10 meals into Firestore. You will see them appear in your Firestore console under the `meals` collection in real time.

## Notes

- All Firestore reads use real-time `snapshots()` streams — the UI updates instantly without any manual refresh
- Dietary filter state lives in `TabsScreen` and is passed down to Categories, Search, and Favourites simultaneously
- The shopping list uses sanitised Firestore keys derived from ingredient names
- Average meal rating is recalculated server-side in Firestore on every review submission
- `dummy_data.dart` seeds Firestore only once — it checks if the `meals` collection is empty before writing
